# 家庭点餐小程序：开发前你需要准备的东西

这份清单是给第一次开发微信小程序的人看的。你不需要懂代码，只需要按步骤把账号、权限、素材和必要信息准备好。后面我写代码、搭页面、设计数据库时，会根据这些信息来配置项目。

## 一、你先要知道我们要做什么

我们第一版先做一个“本地开发演示版”的微信小程序。

意思是：

- 先能在微信开发者工具里跑起来。
- 先给你和家人测试主要流程。
- 暂时不急着提交微信正式审核。
- 暂时不做支付、外卖配送、价格统计、服务器备案。
- 使用微信原生小程序 + 微信云开发。

你需要准备的核心东西有：

- 微信小程序 AppID
- 微信开发者工具
- 微信云开发环境
- 云开发环境 ID
- 小程序基础信息
- 菜品图片和菜谱资料
- 家庭成员测试微信号

## 二、微信小程序 AppID 在哪里获取

AppID 是微信小程序的身份证。每个小程序都有一个唯一 AppID。

### 如果你已经有小程序账号

1. 打开微信公众平台：
   https://mp.weixin.qq.com/
2. 用小程序管理员微信扫码登录。
3. 登录后，确认左上角显示的是“小程序”，不是“公众号”。
4. 在左侧菜单找到：
   **开发与服务 → 开发管理 → 开发设置**
5. 页面里可以看到：
   **AppID（小程序ID）**
6. 把这个 AppID 复制下来，后面给我。

你可以整理成这样：

```text
小程序 AppID：wx1234567890abcdef
```

### 注意

- AppID 可以给我，用来配置本地项目。
- AppSecret 暂时不要给我。
- AppSecret 比 AppID 敏感很多，第一版本地演示通常不需要。

## 三、微信开发者工具在哪里下载

微信开发者工具是用来打开、预览、调试小程序的官方软件。

### 下载位置

1. 打开微信官方开发工具下载页：
   https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 选择你的电脑版本。
3. Windows 电脑就下载 Windows 稳定版。
4. 安装完成后打开。
5. 用你的微信扫码登录。

### 登录后要确认

打开微信开发者工具后，你要确认：

- 你能扫码登录。
- 你的微信号是这个小程序的管理员或开发者。
- 新建项目时可以选择你的小程序 AppID。

如果你新建项目时看不到小程序，通常说明你的微信号还没有开发权限。

## 四、怎么确认自己有没有开发权限

只有小程序管理员或被添加为开发者的人，才能用微信开发者工具开发这个小程序。

### 查看和添加开发者的位置

1. 打开微信公众平台：
   https://mp.weixin.qq.com/
2. 登录小程序账号。
3. 左侧菜单进入：
   **管理 → 成员管理**
4. 查看你的微信号是否在成员列表里。
5. 如果不在，需要管理员添加你。

### 建议权限

如果是你自己开发和测试，建议你的微信号至少拥有：

- 开发者权限
- 体验者权限
- 云开发权限

如果你就是小程序管理员，一般权限会更完整。

## 五、云开发环境在哪里买/开通

微信云开发不是在普通云服务器里买，它是在微信小程序后台或微信开发者工具里开通的。

它可以提供：

- 云数据库：保存家庭、成员、菜单、订单、采购单。
- 云存储：保存菜品图片。
- 云函数：处理登录、权限、统计、生成采购单等逻辑。

### 开通方式一：在微信开发者工具里开通

1. 打开微信开发者工具。
2. 用 AppID 创建或打开小程序项目。
3. 顶部工具栏找到：
   **云开发**
4. 点击进入云开发控制台。
5. 如果还没开通，会提示你开通。
6. 按提示创建一个环境。

### 开通方式二：在微信公众平台里开通

1. 打开微信公众平台：
   https://mp.weixin.qq.com/
2. 登录小程序账号。
3. 左侧菜单找到：
   **开发与服务 → 云开发**
4. 如果没有开通，按页面提示开通。
5. 创建云开发环境。

### 要不要花钱

微信云开发一般会有免费额度或基础额度，但具体额度以微信当前页面为准。

第一版家庭自用、本地演示、少量测试数据，通常用量很小。

你需要注意：

- 如果页面提示选择套餐，优先选择免费或最低成本方案。
- 不要一上来买很高配置。
- 如果微信页面提示必须付费或认证，再把页面截图或文字给我，我帮你判断。

## 六、云开发环境 ID 在哪里找

云开发环境 ID 是我们代码连接云数据库和云存储时要用的。

### 查找位置

1. 打开微信开发者工具。
2. 打开小程序项目。
3. 点击顶部：
   **云开发**
4. 进入云开发控制台。
5. 左上角或环境设置里，会显示当前环境。
6. 找到类似这样的内容：
   **环境 ID**

格式可能类似：

```text
family-order-abc123
```

你可以整理成：

```text
云开发环境 ID：family-order-abc123
```

你可以新建一个文本，名字比如：

```text
小程序开发资料.txt
```

里面先写这些：

```text
小程序名称：
小程序 AppID：
云开发环境 ID：
是否已经开通云开发：是/否
我的微信号是否是小程序管理员：是/否
是否已安装微信开发者工具：是/否
第一版目标：本地开发演示
```

如果你愿意，也可以加：

```text
第一个家庭名称：
主理人昵称：
测试成员昵称：
常用餐次：早餐、午餐、下午茶、晚餐
采购分类：蔬菜区、肉蛋水产区、调料干货区、冷冻区、日用品区
```

## 八、暂时不要给我的敏感信息

这些东西先不要写进文本，也不要发给我：

- 微信公众平台登录密码
- 微信支付密钥
- AppSecret
- 任何私钥文件
- 身份证信息
- 银行卡信息
- 服务器密码

第一版本地演示通常不需要这些。

如果后面真的需要某个敏感信息，我会明确告诉你它是做什么用的、放在哪里、怎么避免写进代码里。

## 九、菜谱资料你需要怎么准备

第一版菜品由主理人手动录入，所以你可以先准备 5 到 10 道菜做测试。

每道菜建议准备：

```text
菜名：
菜品分类：
口味标签：
菜品图片：
这份配方适合几个人吃：
食材 1：名称、数量、单位、备注、采购分类
食材 2：名称、数量、单位、备注、采购分类
食材 3：名称、数量、单位、备注、采购分类
```

示例：

```text
菜名：番茄炒蛋
菜品分类：家常菜
口味标签：酸甜、快手
菜品图片：番茄炒蛋.jpg
这份配方适合几个人吃：2 人

食材 1：番茄，2，个，选熟一点的，蔬菜区
食材 2：鸡蛋，3，个，无，肉蛋水产区
食材 3：小葱，1，把，买小葱不要大葱，蔬菜区
食材 4：盐，适量，勺，家里一般有，调料干货区
```

## 十、菜品图片怎么准备

你可以先用手机拍，也可以先用临时图片。

建议：

- 每道菜 1 张图。
- 图片尽量清晰。
- 横图、方图都可以，但最好统一。
- 第一版建议统一用方图。
- 文件名尽量写菜名，例如：番茄炒蛋.jpg。

不用一开始就准备很多菜。先准备 5 到 10 道，方便我们把流程跑通。

## 十一、测试家庭和测试成员怎么准备

第一版需要测试“主理人创建家庭，成员加入家庭，多人共同点一顿饭”。

所以最好准备：

- 1 个主理人微信号
- 1 到 3 个成员微信号

如果暂时没有多个微信号，也可以先用开发者工具模拟一部分流程，但真实微信号测试会更接近实际使用。

## 十二、我们第一次开发时，你要怎么配合我

你不需要写代码。

你主要做这些：

1. 把 AppID 和云开发环境 ID 准备好。
2. 确认微信开发者工具能打开项目。
3. 准备几道测试菜。
4. 准备几张菜品图片。
5. 告诉我第一个家庭叫什么。
6. 测试我做出来的页面是不是符合你的家庭使用习惯。

我会负责：

- 搭小程序项目结构。
- 写页面。
- 写数据库结构。
- 写云函数。
- 做权限控制。
- 做点餐、共享餐单、采购单流程。
- 帮你整理每一阶段需要测试什么。

## 十三、开发前最终检查表

开始写代码前，你最好确认下面这些都准备好了：

- [ ] 已有微信小程序 AppID
- [ ] 已安装微信开发者工具
- [ ] 你的微信号能登录微信开发者工具
- [ ] 你的微信号有该小程序开发权限
- [ ] 已开通微信云开发
- [ ] 已拿到云开发环境 ID
- [ ] 已准备第一个家庭名称
- [ ] 已准备 5 到 10 道测试菜
- [ ] 已准备菜品图片
- [ ] 已确定采购分类
- [ ] 已确定第一版只做本地开发演示
